1. System Overview & LLM Layer

PE OS uses a unified LLM abstraction layer that allows swapping between OpenAI and Google Gemini models via environment variables — no code changes needed.

Unified LLM Layer

File: apps/api/src/services/llm.ts

FunctionPurposeDefault ModelUse Case
getChatModel()Primary reasoningGPT-4oAgents, analysis, chat
getFastModel()Quick tasksGPT-4o-miniClassification, sentiment
getExtractionModel()Structured extractionGPT-4-Turbo (temp 0.1)Financial data, parsing
getModel()Custom overrideAnyEscape hatch

Environment variables: LLM_CHAT_PROVIDER, LLM_FAST_PROVIDER (values: openai or gemini)

Agent Architecture Types

PatternDescriptionUsed By
StateGraphLangGraph directed graph with conditional edges. Best for multi-step workflows with retry/correction loops.Financial Extraction, Contact Enrichment, Signal Monitor, Email Drafter
ReActLangGraph prebuilt agent with tools. The AI decides which tools to call based on the question.Deal Chat, Portfolio Chat
ParallelConcurrent data fetching followed by structured LLM synthesis.Meeting Prep

2. Financial Extraction Agent

StateGraph — 4 Nodes

Multi-layer financial statement extraction with self-correction. Takes a PDF or Excel file and produces structured financial statements (Income Statement, Balance Sheet, Cash Flow) with confidence scores and validation.

Upload → Extract → Validate → (pass?) → Store → Done
                          ↘ (fail + retries left) → Self-Correct → Validate ↺ (max 3 loops)

Extraction Layers (prioritized fallback)

PriorityMethodBest ForSource Tag
1xlsx library → CSV → GPT-4o classifierExcel files (.xlsx)gpt4o
2Azure Document IntelligenceComplex CIM layouts (if configured)azure
3pdf-parse → GPT-4o classifierText-rich PDFsgpt4o
4GPT-4o VisionScanned/image-only PDFsvision

Node Details

NodeFileWhat It Does
Extractnodes/extractNode.tsDetects file type, routes to best extraction layer, normalizes line items, corrects projected/historical labels, auto-calculates EBITDA if missing
Validatenodes/validateNode.tsMath checks: Revenue - COGS = Gross Profit, Assets = Liabilities + Equity, Cash Flow reconciliation. Flags low-confidence periods.
Self-Correctnodes/selfCorrectNode.tsRe-extracts ONLY the failing statements/periods with a targeted prompt. Merges corrections into existing data.
Storenodes/storeNode.tsSaves to DB, detects merge conflicts (multi-document), sets isActive flags.

Key Files

  • apps/api/src/services/agents/financialAgent/index.ts — Entry point: runFinancialAgent()
  • apps/api/src/services/agents/financialAgent/graph.ts — StateGraph wiring + conditional edges
  • apps/api/src/services/agents/financialAgent/state.ts — State schema (AgentStep[], retryCount, etc.)
  • apps/api/src/services/financialClassifier.ts — GPT-4o classifier, normalizer, derived fields calculator

Post-Processing (Session 37 Additions)

  • normalizeLineItems() — Rounds financial values to 4 decimals, percentages to 2
  • correctPeriodTypes() — Fixes projected vs historical labels (future years → Projected, detects E/F/Est suffixes)
  • computeDerivedFields() — Auto-calculates EBITDA, Gross Profit, and margin % when source doc is missing them

3. Deal Chat Agent

ReAct Agent — 6 Tools

Conversational AI assistant on each deal page. Instead of stuffing all deal data into the system prompt, the agent fetches data on-demand using tools — keeping responses focused and token-efficient.

User asks question → Agent reasons → Selects tool(s) → Fetches data → Generates response

Tools

ToolWhat It Does
search_documentsSemantic search across deal documents via RAG (vector embeddings), falls back to keyword search
get_deal_financialsFetches extracted financial statements + key metrics (revenue, EBITDA, growth)
compare_dealsCompares current deal vs another deal or portfolio averages, shows percentile rankings
get_deal_activityFetches 15 most recent activities (notes, stage changes, uploads, meetings)
update_deal_fieldUpdates deal fields: leadPartner, analyst, source, priority, industry, description + team assignment
suggest_actionReturns navigation suggestions (open VDR, create memo, view financials, change stage)

Key Files

  • apps/api/src/services/agents/dealChatAgent/index.ts — Agent entry point
  • apps/api/src/services/agents/dealChatAgent/tools.ts — All 6 tool definitions
  • apps/api/src/routes/deals-chat-ai.ts — API route, saves messages to DB

Endpoint: POST /deals/:dealId/chat

5. Contact Enrichment Agent

StateGraph — 4 Nodes

AI-powered contact research and profile enrichment. Generates LinkedIn profiles, company details, expertise areas, and deal relevance scores. Routes low-confidence results to human review.

Contact selected → Research → Validate → (confidence ≥ 70%?) → Save → Done
                                               ↘ (< 70%) → Review → Flag for human

Enriched Data Output

  • LinkedIn URL, updated title, company, industry, location
  • Professional bio, expertise areas, connections count
  • Deal relevance score (0-100) with explanation
  • Confidence score based on data completeness
  • Source citations

Confidence Scoring

Data PointPoints
LinkedIn URL found+10
Title updated+5
Company updated+5
Bio generated+5
Expertise areas found+5

File: apps/api/src/services/agents/contactEnrichment/index.ts
Endpoint: POST /api/ai/enrich-contact

6. Meeting Prep Agent

Parallel Fetch + LLM

Generates comprehensive meeting briefs by pulling data from multiple sources in parallel, then synthesizing with GPT-4o structured output.

Data Sources (fetched in parallel)

  1. Deal details + financials + team members
  2. Contact history + past interactions (if contact specified)
  3. RAG document search (if meeting topic specified)
  4. Recent deal activities

Output: Meeting Brief

SectionContent
HeadlineOne-line summary of the deal + meeting context
Deal Summary2-3 sentences on current status, stage, key metrics
Contact ProfileWho you're meeting, their background, relevance
Key Talking Points5-7 data-backed discussion points
Questions to Ask3-5 targeted questions based on gaps in information
Risks to Address2-3 risk areas to probe during the meeting
Document HighlightsKey findings from uploaded documents
Suggested Agenda4-6 structured agenda items with time allocation

File: apps/api/src/services/agents/meetingPrep/index.ts
Endpoint: POST /api/ai/meeting-prep
Input: { dealId, contactId?, meetingTopic?, meetingDate? }

8. Smart Email Drafter

StateGraph — 4 Nodes

Generates professional deal emails with template selection, tone control, and compliance checking. Every email goes through tone and compliance gates before being marked ready.

Select template + tone → Write Draft → Tone Check → Compliance Check → Finalize
                                             ↓                  ↓
                                       Score ≥ 80 ✓      No issues → ready_for_review
                                       Score < 80 → adjusts  Issues found → compliance_issues

Email Templates (7)

#TemplateUse Case
1Initial OutreachFirst contact — intro, why interested, firm background
2Follow UpPost-meeting — thank you, key takeaways, next steps
3Document RequestSpecific docs needed, format preferences, timeline
4LOI IntroductionReference discussions, outline terms, commitment
5Deal UpdateStatus report — developments, decisions, timeline
6Meeting RequestProposed times, attendees, agenda items
7Thank YouGratitude, reference interaction, future opportunity

Tone Options

professional (default)  |  friendly  |  formal  |  direct  |  warm

Compliance Checks

  • MNPI (Material Non-Public Information) disclosure
  • Promissory or commitment language
  • Forward-looking statement warnings
  • FCPA (Foreign Corrupt Practices Act) concerns
  • Confidentiality breach risks

File: apps/api/src/services/agents/emailDrafter/index.ts
Endpoints: POST /api/ai/draft-email, GET /api/ai/email-templates

9. Financial Analysis Suite

13+ analysis modules that run on extracted financial data. Pure math calculations (no AI calls) except for narrative insights and red flag descriptions.

Analysis Modules

ModuleWhat It Calculates
Quality of Earnings (QoE)Earnings quality score, adjustment flags, sustainability assessment
Financial Ratios (18)Growth, profitability, leverage, liquidity, efficiency metrics across 4 tabs
DuPont AnalysisROE decomposition: Profit Margin × Asset Turnover × Equity Multiplier
EBITDA BridgeRevenue → Gross Profit → EBITDA waterfall with period-over-period deltas
Revenue QualityRecurring vs one-time, concentration risk, growth sustainability
Cash Flow AnalysisOperating, investing, financing breakdown, free cash flow, cash conversion
Working CapitalNormalized WC, DSO/DIO/DPO, cash conversion cycle, seasonality
Cost StructureFixed vs variable split, operating leverage, cost trends
Debt CapacityEBITDA-based debt capacity, coverage ratios, recommended leverage
LBO Screen12 scenarios varying entry multiple, leverage, growth, and exit multiple
Red Flag DetectionAutomated flags: margin anomalies, cash flow divergence, leverage spikes
Cross-Doc VerificationCompares values across multiple uploaded documents, flags discrepancies
Portfolio BenchmarkingCurrent deal vs portfolio averages, percentile rankings

Frontend Tabs

The analysis renders below Financial Statements on the deal page with 6 tabs:

  1. Overview — QoE score, key ratios dashboard, EBITDA bridge chart
  2. Deep Dive — 18 ratios with period charts, DuPont, revenue quality
  3. Cash & Capital — Cash flow, working capital, cost structure
  4. Valuation — Debt capacity, LBO screen (12 scenarios)
  5. Diligence — Red flags, cross-doc verification
  6. AI Insights — GPT-4o narrative insights with benchmarking

Key Files

  • apps/api/src/services/analysis/ — All analysis logic (6 files)
  • apps/api/src/routes/financials-analysis.ts — API endpoints
  • apps/web/js/analysis.js — Main orchestrator (tab renderer)
  • apps/web/js/analysis-modules.js — Individual module renderers (14 functions)
  • apps/web/js/analysis-charts.js — Chart.js chart renderers

Endpoints:
GET /deals/:dealId/financials/analysis — Returns all ratio/QoE data
GET /deals/:dealId/financials/insights — Returns AI narrative insights

Architecture Decision: All agents wrap existing service functions — no extraction, validation, or storage logic was duplicated. The agents act as orchestration layers that call into the existing codebase, making them easy to swap out or extend without touching business logic.
