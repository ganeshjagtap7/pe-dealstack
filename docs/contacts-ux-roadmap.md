# PE Dealstack — Contacts/CRM Page Roadmap

*All file paths relative to `/Users/aditya/Desktop/CodeFiles/pe-dealstack`.*

## 1. Diagnosis

Three root themes explain nearly all 31 findings:

1. **AI fires into the void.** Enrichment, follow-up suggestions, and email summaries run fire-and-forget, are never persisted (`contactEnrichment/nodes.ts:385-428` discards the enrichment blob), and produce display-only text with no inline actions — so the user pays 15-min agent runs repeatedly and can't act on the output.
2. **The list view is missing table stakes.** No select-all, bulk edit, dedupe, staleness filters, or saved views — exactly the loudest complaints about category leader Affinity ([westleydang.com/affinity](https://westleydang.com/affinity/)). Meanwhile Gmail suggestions and CSV import both create contacts with zero duplicate checking (`apps/api/src/routes/contacts.ts:232-297, 374-423`), so the data is degrading as the ingestion features succeed.
3. **Contacts and deals are one-way silos.** Deal pages never show external contacts (`apps/api/src/routes/deals-list.ts:138-170`), emails auto-link contacts to deals but never write back interactions, and follow-up reminders only fire when the user happens to load /contacts (`apps/api/src/routes/notifications.ts:590-617`) — the relationship loop never closes.

## 5. Suggested sequencing — first 5 moves

1. **Unique-email constraint + dedupe checks (QW-1).** Do this before anything else: Gmail suggestions and CSV import are actively manufacturing duplicates, and every later feature (merge, smart fields, cadence resets) keys off email identity. Cheapest now, exponentially costlier later.
2. **Persist enrichment + actionable outputs (MB-2 + QW-6/7).** Stops burning LLM spend on regeneration, gives every AI surface a durable substrate (`enrichmentData`), and establishes the provenance pattern (badge + never overwrite manual edits) that all future AI features reuse.
3. **Follow-up loop: cron + mark-done + note populate (MB-1 + QW-2).** The single highest-impact finding (5). Turns follow-ups from a contacts-page curiosity into a trustworthy system — which is what makes the later cadence/digest big bet credible.
4. **Key Contacts on the deal page (MB-3) + stale surfacing (QW-3/4).** First visible cross-surface payoff: deals show who's involved and who's gone cold, using data that now exists from moves 1-3. This is the feature a PE user demos to a colleague.
5. **Bulk ops + activity filters (MB-4 + MB-7).** With clean, deduped, enriched data flowing in, the list finally needs to handle volume — and select-all + "not contacted in 60 days" converts the page from a rolodex into a working pipeline tool, setting up saved views (BB-4) and the import overhaul next.

Why this order compounds: data integrity (1) → durable AI substrate (2) → a reminder loop users trust (3) → cross-surface visibility that proves value (4) → scale ergonomics (5). Each big bet then lands on foundations already shipped rather than requiring its own plumbing.
