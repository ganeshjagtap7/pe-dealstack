Hi Pushkar — thanks for the detailed testing! Below is a point-by-point response to every issue you flagged. All fixes are done and ready for re-testing.

1. Organization Isolation / New Signup Issues

FIXED New signup sees other firm's deals and contacts
You reported: "THE DEAL PAGE IS NOT EMPTY" / "CONTACTS PAGE IS NOT EMPTY" / "IS NOT ACCURATE IN BOTH DEVICES SHOWS 18 DEALS"
What happened: When you signed up with drxpsus@gmail.com, the system had a timing issue — your user profile wasn't created fast enough, so the app loaded without knowing which organization you belong to. It showed everyone's data instead of filtering to just your firm.
Fix: The system now auto-creates your user profile and organization on the very first API call after signup. No more timing issues. New signups will see empty pages as expected.
FIXED Can't create deals
You reported: "DEAL IS NOT GETTING CREATED" (Ingest Deal Data showed "Upload Failed")
Same root cause — the system didn't know your organization ID, so it couldn't save the deal. Fixed by the same auto-create fix above.
FIXED Can't create contacts
You reported: "DID NOT GET CREATED SHOWED ISSUE" — error: "null value in column organizationId"
Same root cause. Now that org ID is always available, contact creation works.
FIXED Settings page not loading / profile not updating
You reported: "ITS NOT UPDATING MY PROFILE" — settings page showed "Loading..." forever
Same root cause. The settings page calls /api/users/me which was failing for new users. Now works on first load.

2. Document & Data Room Issues

FIXED Uploaded documents disappear from Data Room
You reported: "THE FILE IS VISIBLE IN RECENT DOCUMENTS BUT ITS NOT VISIBLE IN DATA ROOM" / "DOCUMENT SHOWED PROCESSING AND WHEN I WENT BACK THE DOCUMENT DIDNT SHOW UP" / "DOCUMENT DISAPPEARED FROM THE CHAT TOO"
What happened: Documents need to be assigned to a folder to show up in the Data Room. New deals didn't have any folders, so uploaded files had no folder = invisible.
Fix: When you upload a file to a deal that has no folders yet, the system now automatically creates the default folder structure (Financials, Legal, Commercial, HR & Data, IP) and puts the file in the right folder. Files uploaded via AI chat also get assigned to a folder now.
FIXED Can't extract financials from Data Room files
You reported: "ONLY EXTRACTION WORKS VIA INGEST DEAL DATA AND NOT VIA DATA ROOM" / "DATAROOM FILES AS OFF CURRENTLY CANT BE EXTRACTED"
Fix: Added an "Extract Financials" option to the file menu (three-dot menu) in the Data Room. You can now right-click any PDF or Excel file in the Data Room and trigger extraction directly from there.

3. Financial Extraction Issues

FIXED Decimal accuracy is wrong
You reported: "EXTRACTION THE NUMBER IS IN MILLION EVEN THOUGH ITS IN MILLION THE DECIMAL ITS CHANGING TOO IS VERY WRONG"
What happened: The AI was returning numbers with floating-point artifacts (like 12.50000001 instead of 12.5).
Fix: All extracted values are now rounded — financial values to 4 decimal places, percentages to 2 decimal places. Numbers display cleanly now.
FIXED Projected vs Historical year detection is wrong
You reported: "FOR PROJECTED PERIODS FIRST IT ASSUMED 2024 TO BE PROJECTED" / "PROBABLY NO LOGIC ON HOW TO DEFINE PROJECTED OR HISTORICAL YEARS"
Fix: Added a verification step after extraction. If a year is in the future (e.g., 2027, 2028) but the AI labeled it as "Historical", the system now corrects it to "Projected". Also detects suffixes like "E", "F", "Est" in period names. Past years won't be incorrectly marked as projected.
FIXED EBITDA not showing when source doc doesn't have it
You reported: "THE FINANCIALS I ADDED DID NOT HAVE A SEPARATE EBITDA NO. BUT IT CAN BE CALCULATED"
Fix: The system now auto-calculates derived metrics when they're missing:
  • EBITDA = EBIT + D&A (or Revenue - COGS - Total OpEx)
  • Gross Profit = Revenue - COGS
  • Gross Margin % and EBITDA Margin % from the above
These only fill in gaps — if the AI already extracted the value, it won't be overwritten.
FIXED Cell editing fails — "Could not save the change"
You reported: "WHENEVER I TRY TO MAKE CHANGES IT SHOWS COULD NOT SAVE THE CHANGE"
What happened: Two bugs — (1) the save was using the wrong user ID format which caused a database error, and (2) editing one cell was accidentally wiping all other cells in that row.
Fix: Fixed the user ID lookup, and changed from full-row replacement to single-cell update. Editing now works correctly.
FIXED Balance Sheet chart missing on some deals
You reported: "WORKING PROPERLY BUT IT SHOWED FOR ONE OUT OF TWO FINANCIALS I UPLOADED — CHECK THE DEALS BUFFER AND PRE FLOP WIZARD"
What happened: When Balance Sheet data exists but the key chart values (cash, receivables, equity, etc.) are all empty, the chart showed as a blank white space.
Fix: Now shows a helpful message explaining that the data exists but chart values aren't extracted yet, and suggests re-extracting.

4. Team Invitation

FIXED Invitation email not getting sent
You reported: "THE EMAIL INVITE IS NOT GETTING SENT"
What happened: The email service needs a verified domain to send to external addresses. The invitation was being created in the database but the email wasn't going out.
Fix: When email delivery fails, the app now shows a "Copy Invite Link" button so you can copy the invitation link and share it manually (via WhatsApp, Slack, email, etc.). The link works the same way — recipient clicks it to accept the invite.

5. Contacts Page UI

Fix: Updated the CSS build to scan all pages. Grid/List toggle is now visible.
FIXED "More" dropdown not visible (Export/Import)
You reported: "CANT SEE MORE OPTION TO EXPORT THE FILE"
Same root cause as above. The "More" button with Export to CSV, Import, and Group by Company options is now visible.
